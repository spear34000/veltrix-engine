Veltrix Engine Architecture

v0.1 Development 2026-06-13

Veltrix is a lightweight, dependency-free inference engine for transformer-based LLMs, targeting edge devices (N100, phones) with 40+ tok/s for Q4_0 1B-class models.

Introduction

Goal: Extend Veltrix to a multi-format inference engine with zero external deps, targeting 40+ tok/s on N100/phone for Q4_0 1B-class models.

Core Principles:

  • Zero external dependencies — JSON parser, protobuf reader all self-contained
  • Drop-in compatibility — Place any popular GGUF model in models/ and run
  • Multi-format — GGUF, Safetensors, ONNX
  • Multi-architecture — Qwen2, Llama, Mistral, Gemma, Phi-2, Phi-3, StarCoder2
  • Optimized for edge — AVX2 SIMD, quantized GEMV, memory consolidation
  • Predictive optimization — Free-energy scheduler, meta-learning, speculative decoding

Development History

Phase 1 — Format Abstraction DONE

Created include/format.h and src/format.c with the loader registry pattern, format auto-detection, weight resolution by name pattern, and architecture auto-configuration. Three format loaders registered: GGUF, Safetensors, ONNX.

Phase 2 — Safetensors Loader DONE

Embedded JSON parser (js_skip_ws, js_skip_string, js_skip_value, js_find_key), HF config.json reader, F16/BF16→F32 conversion.

Phase 3 — Tokenizer DONE

BPE, SentencePiece, TikToken implementations. Auto-load from GGUF metadata or HF tokenizer.json.

Phase 4 — ONNX Loader DONE

Minimal protobuf wire format (pb_read_varint, pb_read_tag, pb_read_string), synthetic test passes.

Phase 5 — LoRA Adapter DONE

Safetensors-based LoRA loader with vx_lora_load, vx_lora_merge (W += α/r · B × A), vx_lora_destroy.

Step 1 — Hot-path malloc/free Removal DONE

Permanent scratch buffer on vx_model struct, norm_tmp reuse via scratch pointer offset, no malloc/free in vx_model_forward hot path.

Step 2 — SIMD Kernels DONE

AVX2 dispatch in tensor.c: rms_norm, softmax, silu, add_f32, mul_f32, gelu, rope. ONNX test: 971→872 µs (~10% improvement).

Step 3 — Direct Q4_0 GEMV DONE

vx_gemv_q4_0_avx2 and sparse variants replace dequant+gemv path for Q4_0 weights. Fixed fp16 bug (vx_fp16_to_fp32).

Step 4 — Memory Layout DONE

Single contiguous tensor_data allocation consolidating all weight tensors (reduces TLB misses, improves prefetching).

Step 5 — Dequant Scratch Buffer DONE

Permanent dequant_tmp buffer on vx_model, eliminates 256 MB malloc/free per forward for Q6_K tensors. Passed to all apply_gemv/apply_gemv_sparse_rows calls.

Current Problems OPEN

1. Tokenizer/Model vocab mismatch (Qwen2): GGUF file has no vocab_size KV pair (vocab size = 151936 from tokenizer.ggml.tokens array length). Code falls back to 32000, truncating the tokenizer to 32000 tokens. model.config.n_vocab is auto-corrected to 151936 from tensor shape, but tokenizer stays at 32000.

2. Poor output quality: Downstream of tokenizer mismatch — token IDs 32000+ are not in the tokenizer vocabulary, causing decode failures and gibberish output.

3. Throughput bottleneck: TinyLlama 1.1B only 4.6 tok/s (target 40+). Q6_K output.weight previously causing 256 MB malloc per forward (now fixed with dequant_tmp). Further investigation needed.

System Architecture

The engine is organized as a pipeline with clear separation of concerns:

┌──────────────┐    ┌──────────────┐    ┌──────────────┐    ┌──────────────┐
│  Format      │ →  │  Weight      │ →  │  Memory      │ →  │  Forward     │
│  Detection   │    │  Resolution  │    │  Consolidation│    │  Engine      │
│  (magic/ext) │    │  (name match)│    │  (one alloc) │    │  (inference) │
└──────────────┘    └──────────────┘    └──────────────┘    └──────────────┘
                                                                    │
              ┌──────────┐  ┌──────────┐  ┌──────────┐  ┌──────────┐│
              │ AVX2     │  │ Q4_0     │  │ Dequant  │  │ Sparse   ││
              │ SIMD     │  │ Direct   │  │ Scratch  │  │ GEMV     ││
              │ Kernels  │  │ GEMV     │  │ Buffer   │  │ (gate)   ││
              └──────────┘  └──────────┘  └──────────┘  └──────────┘▼

Key Design Decisions

  • Architecture auto-detection via general.architecture string; per-arch defaults in arch_table in format.c.
  • Tensor dimensions stored in ne[0..3] following GGUF convention; ONNX/safetensors dims reversed on load.
  • No constructor attributes for loaders; explicit vx_format_init() called from vx_model_create.
  • Tokenizer auto-loaded during model creation (best-effort, non-fatal on failure).
  • LoRA merge performed in-place on model weights (pre-merge) rather than on-the-fly.
  • Performance improvement order: hot-path malloc/free → SIMD → direct quantized GEMV → memory layout → dequant scratch.

Model Lifecycle

Creation: vx_model_create(path, &model)

1. vx_format_init()              → register loaders
2. fopen(path), read 16 bytes    → detect format (GGUF magic / Safetensors / ONNX)
3. loader → load(path, model)    → format-specific loading
   ├── GGUF: parse header → KV pairs → tensor info → tensor data
   │          → vx_model_apply_arch() for MLP/norm/bias
   ├── Safetensors: read config.json, JSON header, tensor data
   │          → reverse dims (HF → Veltrix), F16/BF16→F32
   └── ONNX: parse protobuf ModelProto → GraphProto → initializer
4. vx_model_resolve_weights()    → match tensor names to model fields
5. Memory consolidation          → single contiguous allocation
6. KV cache init                 → calloc(n_layers × max_seq_len × ...)
7. Tokenizer auto-load           → GGUF metadata or tokenizer.json
8. Meta-system + FE state init   → if enabled
9. dequant_tmp allocation        → largest tensor size

Destruction: vx_model_destroy(model)

free(tensor_data)
free(k_cache), free(v_cache)
free(scratch), free(dequant_tmp)
free(tensors[])
destroy meta-predictors
free(tokenizer)
free(model)

Forward Pass

vx_model_forward(model, tokens, n_tokens, logits)
│
├── 1. EMBEDDING
│     x[i] = tok_embd[tokens[i]]  for i=0..n_tokens-1
│
├── 2. FOR EACH LAYER l = 0..n_layers-1:
│     │
│     ├── a) Meta decision (if enabled)
│     │     skip / exact / exit
│     │
│     ├── b) Pre-attention norm
│     │     x = RMSNorm(x, attn_norm[l].weight)
│     │
│     ├── c) QKV projections
│     │     q = apply_gemv(attn_q[l], x, n_embd, n_embd, dequant_tmp)
│     │     k = apply_gemv(attn_k[l], x, kv_dim, n_embd, dequant_tmp)
│     │     v = apply_gemv(attn_v[l], x, kv_dim, n_embd, dequant_tmp)
│     │     [+ bias if present]
│     │
│     ├── d) RoPE
│     │     vx_rope(q, k, pos, n_heads, n_kv_heads, head_dim, rope_dims, theta)
│     │
│     ├── e) KV cache store (pos = cache_len)
│     │
│     ├── f) Attention
│     │     scores = q · k_cache^T / sqrt(head_dim)
│     │     causal mask + softmax
│     │     attn_out = scores · v_cache
│     │     apply_gemv(attn_o[l], attn_out, n_embd, n_embd, dequant_tmp)
│     │
│     ├── g) Residual
│     │     x = x + attn_out
│     │
│     ├── h) FFN norm
│     │     x = RMSNorm(x, ffn_norm[l].weight)
│     │
│     ├── i) FFN
│     │     SwiGLU: out = SiLU(gate·x) * (up·x)  × down
│     │     GEGLU:  out = GELU(gate·x) * (up·x)  × down
│     │     CLASSIC: out = GELU(up·x)   × down
│     │
│     ├── j) Residual
│     │     x = x + ffn_out
│     │
│     └── k) Meta update (if enabled)
│
├── 3. Final norm
│     x = RMSNorm(x, output_norm.weight) [+ bias]
│
├── 4. Output projection
│     logits = x × output_weight.T  (or tok_embd.T if tied)
│
└── 5. cache_len += n_tokens

GEMV Dispatch

apply_gemv(w, x, y, rows, cols, dqtmp):
  switch w->type:
    VX_TYPE_F32:
      vx_gemv_f32_avx2(y, (float*)w->data, x, rows, cols)    #if AVX2
      vx_gemv_f32(...)                                         #else

    VX_TYPE_Q4_0:
      vx_gemv_q4_0_avx2(w->data, x, y, rows, cols)            #if AVX2 (direct)
      dequant → f32_gemv                                        #else

    default (Q6_K, Q2_K, etc.):
      tmp = dqtmp ? dqtmp : malloc(rows * cols * sizeof(float))
      vx_dequantize(w->data, tmp, rows * cols, w->type)
      vx_gemv_f32_avx2(y, tmp, x, rows, cols)                  #if AVX2
      vx_gemv_f32(...)                                          #else
      if (!dqtmp) free(tmp)

Vocab Size Bug (Qwen2)

Qwen2.5 GGUF files have no vocab_size metadata key. Vocab size (=151936) is only available from the length of tokenizer.ggml.tokens string array (151936 entries) and from token_embd.weight tensor shape. The tokenizer loader falls back to 32000, truncating. The model config is auto-corrected by tensor shape detection, but the tokenizer stays truncated.

Quantization

Block structures for Q4_0 through Q6_K defined in quantize.h. Full dequantize implemented for Q4_0, Q4_1, Q5_0, Q5_1, Q8_0, Q2_K, Q6_K. Direct GEMV (no dequant) for Q4_0 with AVX2. Other types fall through to dequant+gemv path.

Meta-Learning

2-layer MLP (n_embd → 128 → n_embd) with ReLU hidden activation, trained via online SGD during inference. Predicts hidden state norms, attention patterns, and early exit decisions.

Meta-Predictor

predict(hidden_state):
  h = ReLU(W1 · x + b1)
  y = W2 · h + b2
  confidence = precision / (precision + ||y - actual||²)

update(hidden_state, actual):
  error = y - actual
  W1 -= lr * error · W2ᵀ · ReLU'(h) · xᵀ
  W2 -= lr · h · errorᵀ
  update running_mean / running_var

Decisions

  • Layer skip: predict layer output norm; if confidence > threshold, skip computation
  • Attention pattern: predict attention distribution; skip if confidently similar to previous
  • Early exit: predict if early layer output is sufficient

Free-Energy Scheduler

Computes free energy and prediction error at each layer to decide compute level (EXACT / APPROX / SKIP).

Free Energy = 0.5 × precision × ||predicted - actual||² - 0.5 × ln(precision)
Prediction Error = MSE(predicted, actual)

Policies

  • MIN_FE: minimize total free energy
  • TARGET_SKIP: target a specific skip rate
  • ADAPTIVE: adapt based on recent history

LoRA Adapters

Loads LoRA safetensors and adapter_config.json, applies W += α/r · B × A to matched weight tensors.

Limitations

  • Only supports linear projections (no embedding, no Conv2D)
  • Merge is destructive (permanent modification of model weights)
  • No support for DoRA, PiSSA, or other PEFT variants

Known Issues

Tokenizer Vocab Mismatch BUG

File: src/tokenizer.c:700-715, src/gguf.c:484-486

Symptom: Qwen2.5 GGUF models show "Tokenizer: BPE (32000 tokens, bos=1 eos=2)" while model vocab is 151936. Output text is garbled for tokens > 32000.

Root cause: GGUF files generated by llama.cpp's convert script for Qwen2.5 do not include a vocab_size metadata key. The vocabulary size is only available implicitly from the length of tokenizer.ggml.tokens array and from tensor dimensions. The C code assumes a vocab_size KV pair exists, falls back to 32000 when missing, and gguf_scan_str_array truncates to this incorrect limit.

Fix: When vocab_size KV is missing, derive it from tokenizer.ggml.tokens array length or from token_embd.weight tensor dimensions.

K-Quant Type Sizes BUG

File: src/tensor.c:22-24

vx_type_size() returns 0 for Q4_K, Q5_K, Q8_K types. Only Q3_K and Q6_K have correct sizes.

Missing K-Quant Dequant BUG

File: src/quantize.c

Only Q2_K and Q6_K dequant implementations exist. Q3_K, Q4_K, Q5_K, Q8_K dequant functions declared in header but not implemented.

FE State Reset BUG

File: src/scheduler.c:30-32

vx_fe_state_reset() only resets 1 layer (sizeof(float) * 1 instead of * n_layers).

gguf_scan_str_array Offset Leak BUG

File: src/tokenizer.c:517-534

When array read is truncated (n < alen), remaining items are not skipped, leaving offset pointing mid-array. Currently masked because callers reset kv_off = 24 before each scan.

Throughput PERF

TinyLlama 1.1B Q4_0: 4.6 tok/s on N100 (target 40+). Suspected causes: (1) Q6_K output.weight still goes through dequant path despite dequant_tmp, (2) bandwidth utilization only ~10% (2.9 GB/s vs 30 GB/s theoretical).

Missing GGUF Type Mapping BUG

gguf_to_vx_type() has no cases for GGML_TYPE_Q4_1, Q5_0, Q5_1, Q8_1 (returns F32 fallback).
